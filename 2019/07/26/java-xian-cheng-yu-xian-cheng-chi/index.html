Java线程与线程池

发表于 2019-07-26 更新于 2019-08-06 分类于 Java并发编程

在操作系统中，进程和线程是很常见的概念，其中：进程是资源分配的基本单位，而线程是处理器调度的基本单位。比如在一个Java进程中，就有多个线程：垃圾收集线程、main线程等。一个进程内的线程可以共享该进程拥有的资源，可以很容易的实现相互之间的通信；并且线程的切换也比进程的切换开销要小的多，所以对于相似的任务，可以将它们安排在同一个进程中的不同线程上，这样会比以多进程的方式执行的效率高。

1.创建并使用线程

在Java中，有3种方式创建和使用线程：

  • 继承Thread类
  • 实现Runnable接口
  • 实现Callable接口

1.1继承Thread类

  Thread类的声明为:public class Thread implements Runnable,通过继承Thread类并重写其中的public void run()方法，该方法定义了线程执行的逻辑，再调用start()方法就可以启动一个线程了。Thread类以及Runnable接口源码如下：

@FunctionalInterface
public interface Runnable {
    //这个方法就是线程要执行的逻辑
    public abstract void run();
}

public
class Thread implements Runnable {
    //实现了Runnable接口的类的实例，Thread本身就是
    private Runnable target;

    //重写了Runnable接口中的run()方法
    @Override
    public void run() {
        if (target != null) {
            target.run();
        }
    }

    /**
     * Causes this thread to begin execution; the Java Virtual Machine
     * calls the <code>run</code> method of this thread.
     * <p>
     * The result is that two threads are running concurrently: the
     * current thread (which returns from the call to the
     * <code>start</code> method) and the other thread (which executes its
     * <code>run</code> method).
     * <p>
     * It is never legal to start a thread more than once.
     * In particular, a thread may not be restarted once it has completed
     * execution.
     *
     * @exception  IllegalThreadStateException  if the thread was already
     *               started.
     * @see        #run()
     * @see        #stop()
     */
    public synchronized void start() {

        if (threadStatus != 0)
            throw new IllegalThreadStateException();

        group.add(this);

        boolean started = false;
        try {
            start0();
            started = true;
        } finally {
            try {
                if (!started) {
                    group.threadStartFailed(this);
                }
            } catch (Throwable ignore) {
                /* do nothing. If start0 threw a Throwable then
                  it will be passed up the call stack */
            }
        }
    }

    private native void start0();
}

源码中对start()方法的注释说的很明白：调用start()方法会创建一个新的线程去执行该Thread对象的run()方法；但是在start()方法中并没有出现run()，应该是放到native的start0()中实现了。
再看run()方法，其中的target就是成员变量private Runnable target;,如果采用继承Thread类重写run()方法来创建线程，那么target = null，所以新建的线程会调用我们重写的run()方法。父类调用被子类重写的方法，模版模式。
Thread类中有关线程操作的方法：

方法名称 说明
void join() 等待该方法所属的线程执行完毕，就是说在a线程中调用b.join()，则a线程会被挂起，直到b线程执行结束
static native void sleep(long millis) 休眠当前正在执行的线程，时间为millis毫秒
static native void yield() 向线程调度器建议让其它线程运行，但只是建议
static native Thread currentThread() 返回当前正在执行的线程对象
void interrupt() 中断某一线程，如果该线程处于阻塞、等待状态，会抛出InterruptedException异常，从而提前结束该线程。但是不能中断 I/O 阻塞和 synchronized 锁阻塞

Object类中关于线程操作的方法：

方法名称 说明
void wait() 调用wait()使得线程等待锁并挂起，直到别的线程调用notify()或notifyAll()唤醒此锁上挂起的线程
native void wait(long timeout) 在wait()方法的基础上增加了超时的功能
void wait(long timeout, int nanos) 计时精确到纳秒级别
native void notify() 唤醒一个在某个锁上等待的线程
native void notifyAll() 唤醒所有在某个锁上等待的线程
  • wait()和notify()应该应用于synchronized修饰的方法或代码块中，因为sychronized即是对某一对象加锁。
  • wait()和sleep()的区别：wait()会释放锁，而sleep()不会。

1.2创建Runnable接口的实现类，并使用new Thread(Runnable target)构造函数

先创建一个Runnable接口的实现类，然后使用Thread类的public Thread(Runnable target)构造函数，另外几个构造函数是给新创建的线程设置线程名、线程组等信息。
同样，还是需要start()方法来启动线程将其变为就绪态，此时，新创建的Thread对象中的target就不为空了，所以就会执行target也就是Runnable实现类中的run()方法了。

1.3实现Callable接口，并使用new Thread(Runnable target)构造函数

  Callable接口：

    @FunctionalInterface
    public interface Callable<V> {
        V call() throws Exception;
    }

Callable接口类似于Runnable接口，但Runnable中的run()方法没有返回值，而Callable中的call()方法有返回值。
但是Thread类中并没有参数中含有Callable实现类对象的构造函数。
  Future接口：

    public interface Future<V> {

        boolean cancel(boolean mayInterruptIfRunning);

        boolean isCancelled();

        boolean isDone();

        V get() throws InterruptedException, ExecutionException;

        V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException;
    }

看源码中的注释：Future接口代表异步计算的结果，其中定义的这几个抽象方法是用于判断执行的状态以及获取执行的结果等。
  RunnableFuture接口：

    public interface RunnableFuture<V> extends Runnable, Future<V> {
        void run();
}

从名字上也可以看出它应该是与Runnable接口和Future接口都有关系。
  FutureTask类：

    public class FutureTask<V> implements RunnableFuture<V> {

        public FutureTask(Callable<V> callable) {
        if (callable == null)
            throw new NullPointerException();
        this.callable = callable;
        this.state = NEW;       // ensure visibility of callable
        }

        public void run() {
        if (state != NEW ||
            !UNSAFE.compareAndSwapObject(this, runnerOffset,
                                         null, Thread.currentThread()))
            return;
        try {
            Callable<V> c = callable;
            if (c != null && state == NEW) {
                V result;
                boolean ran;
                try {
                    //新创建的线程会执行Callable中的call()方法
                    result = c.call();
                    ran = true;
                } catch (Throwable ex) {
                    result = null;
                    ran = false;
                    setException(ex);
                }
                //记录返回的结果result
                if (ran)
                    set(result);
            }
        } finally {
            // runner must be non-null until state is settled to
            // prevent concurrent calls to run()
            runner = null;
            // state must be re-read after nulling runner to prevent
            // leaked interrupts
            int s = state;
            if (s >= INTERRUPTING)
                handlePossibleCancellationInterrupt(s);
        }
    }
    ...
}

FutureTask类实现了Future接口和Runnable接口，而类中有一个Callable<V>类型的成员变量以及用于接收call()方法返回值的Object类型的成员变量outcome。
所以，通过实现Callable接口创建线程的逻辑，就是

  • 创建Callable的实现类c，实现V call()方法，以此为线程的执行逻辑和返回值
  • 以上一步创建的Callable接口的实现类实例c为参数，创建FutureTask类的实例ft
  • 因为FutureTask类的实例ft实现了Runnable接口，所以以ft为参数，调用Thread类的构造函数new Thread(Runnable target)就能创建一个线程了

启动线程后，线程运行并将结果存入FutureTask实例中，可以调用其中的方法查看运行状态以及运行的结果。

    class MyCallable implements Callable<Integer> {
        public Integer call() {
            return 123;
        }
    }

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        MyCallable mc = new MyCallable();
        FutureTask<Integer> ft = new FutureTask<>(mc);
        Thread thread = new Thread(ft);
        thread.start();
        System.out.println(ft.get());
    }

2.线程的互斥与同步

对于并发执行的各个线程，有时需要保证程序执行的正确性，就必须对各个线程所共享的资源加锁，以实现互斥访问。Java提供了两种锁机制来控制多个线程对共享资源的访问：

  • synchronized（由JVM实现）
  • ReentrantLock（由JDK实现）

2.1 synchronized关键字

该关键字可用于代码块前或方法签名中，它后面括号中的对象就是要加锁的对象。

1.用在代码块前

用法：

    public void func() {
    synchronized (this) {
         ...
    }
}

代码中的this可以是任一普通对象，当进入该代码块的之前，当前线程尝试获取该对象，如果该对象已被其它线程获取，那么当前线程就需要进入阻塞状态直到其它线程退出该代码块。
如果括号中是一个类对象，比如a.class，那么synchronized锁定的是a这个类，两个线程不能同时进入同一个类的不同对象的代码块。

2.用于普通方法签名中

这种用法synchronized锁定的是该方法所在的对象：

    public synchronized void func () {
     ...
    }

3.用于静态方法签名中

这种用法synchronized锁定的是该方法所在的类对象:

    public synchronized static void fun() {
     ...
    }
